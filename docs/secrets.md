# Facto Secrets

Facto V1 keeps build credentials on the trusted Mac worker.

## Storage

Local development:

- Controller env: `.facto/controller.env`
- Worker env: `.facto/worker.env`
- SQLite database: `.facto/controller.sqlite`
- Worker checkouts/artifacts: `.facto/workspaces`

Production worker:

- Worker env: `/opt/facto/secrets/worker.env`
- App Store Connect key file: `/opt/facto/secrets/AuthKey_<KEY_ID>.p8`
- Worker checkouts/artifacts: `/opt/facto/workspaces`

The setup scripts create env files with mode `600`. The `.facto` directory is ignored by git.

## Setup

Create local env files:

```bash
npm run setup:local
```

Create the macOS worker env template:

```bash
bash scripts/setup-worker-macos.sh
```

Run with an env file:

```bash
FACTO_ENV_FILE=.facto/controller.env npm run dev:controller
FACTO_ENV_FILE=.facto/worker.env npm run dev:worker
```

## Required Values

`EXPOFACTO_API_KEY`

Used by local machines, CI, and runners to authenticate with Expo Facto. Create it from the dashboard, then pass it with `--api-key` or export it before running CLI commands.

`FACTO_WORKER_TOKEN`

Used by workers to lease jobs and upload logs. Generated by `npm run setup:local`; copy the same value to `/opt/facto/secrets/worker.env` on the Mac worker.

`PPL_REPO_URL`

The SSH URL for the PPL repository. The worker also needs repository access through either a deploy key in `~/.ssh` or a GitHub token.

`EXPO_TOKEN`

Create this from Expo account access tokens. It lets `eas build --local` and `eas submit` run non-interactively.

`EXPO_PUBLIC_OPEN_MEMORIES_API_URL`

Public runtime URL compiled into the PPL app.

`EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY`

Public Clerk publishable key compiled into the PPL app.

`OPEN_MEMORIES_CLERK_ENV=live`

PPL build selector for the live Clerk environment.

`EXPO_ASC_API_KEY_PATH`, `EXPO_ASC_KEY_ID`, `EXPO_ASC_ISSUER_ID`

Preferred submission credentials. Store the `.p8` file under `/opt/facto/secrets` and keep it out of git.

`EXPO_APPLE_ID`, `EXPO_APPLE_APP_SPECIFIC_PASSWORD`

Temporary fallback for early parity testing if App Store Connect API key submission is not configured yet.

## Repository Access

Prefer a read-only deploy key for the worker:

```bash
npm run setup:github-key
```

Add the printed public key as a read-only deploy key on the GitHub repository. Then set `PPL_REPO_URL` to the generated host alias form, for example `git@github.com-facto-ppl:OWNER/REPO.git`.

## Production Rule

Until encrypted controller-side secret storage exists, do not put Expo, Apple, Clerk, or API credentials into SQLite or job creation payloads. Put them in the worker env file only.
